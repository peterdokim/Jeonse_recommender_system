# 🏠 전세 안심 추천 — 개인화 전세 대안 추천 시스템 기획안

## 부제
**구조 기반 리스크 스코어, 실거래가 데이터, 위험 성향 설문을 결합해 사용자의 보증금과 생활 조건에 맞는 더 안전한 전세 대안을 추천하는 개인화 의사결정 지원 시스템**

---

## 1. 기획 배경

전세 계약을 준비하는 사용자는 시세, 교통, 평형, 생활 인프라 정보는 비교할 수 있지만, 정작 가장 중요한 질문에는 답하기 어렵다.

> **“이 계약에서 내 보증금을 무사히 돌려받을 수 있을까?”**

기존 부동산 서비스는 시세 정보나 매물 조회 중심으로 설계되어 있어, 사용자가 자신의 보증금 규모를 기준으로 **보증금 회수 위험**을 해석하고 비교하기 어렵다. 또한 현재 보고 있는 후보보다 **비슷한 예산과 생활권 안에서 더 안전한 대안이 무엇인지**를 알려주는 도구도 부족하다.

이에 본 프로젝트는 기존의 전세 안전 진단 기능을 확장하여, 단순히 위험도를 보여주는 데서 끝나는 것이 아니라 **사용자의 조건 안에서 현재 후보보다 더 안전한 전세 대안을 추천하는 개인화 추천 시스템**을 제안한다.

---

## 2. 핵심 프레이밍 전환

### 기존 프레이밍
- 전세 안심 검진소
- 위험을 진단해주는 서비스
- “이 집이 위험한가?”에 답하는 도구

### 새로운 프레이밍
- 전세 안심 추천
- 더 안전한 대안을 제시하는 서비스
- **“그래서, 내 조건 안에서 어디를 선택해야 더 안전한가?”**에 답하는 도구

즉 이 서비스의 핵심은 위험 진단 자체가 아니라,  
**현재 후보보다 더 나은 선택지를 제시하는 것**이다.

---

## 3. 제품 한 줄 정의

**사용자의 보증금, 현재 관심 후보, 생활권, 위험 성향을 입력받아, 구조 기반 리스크 점수와 손실 노출 금액, 실거래가 기반 가격 맥락을 함께 반영해 더 안전한 전세 대안을 추천하는 시스템**

---

## 4. 문제 정의

### 4-1. 사용자는 위험을 해석하기 어렵다
전세가율, 하락폭, 사고율 같은 숫자는 볼 수 있지만,  
그 숫자가 실제로 **내 계약에서 얼마나 위험한지** 이해하기 어렵다.

### 4-2. 같은 리스크라도 내 상황에서의 치명도는 다르다
같은 하락률이라도 보증금 2억인 사람과 5억인 사람의 손실 충격은 다르다.  
즉 사용자가 궁금한 것은 “이 집이 위험한가?”보다 **“내 돈 기준으로 얼마나 아픈가?”**이다.

### 4-3. 현재 후보보다 더 나은 대안을 찾기 어렵다
사용자는 보통 이미 보고 있는 동네나 단지가 있고,  
그 후보가 위험할 때 “그럼 어디가 더 낫지?”를 알고 싶어 한다.  
하지만 기존 서비스는 **비슷한 예산·생활권 안에서 더 안전한 대안 추천**을 제공하지 못한다.

### 4-4. 계약 직전에는 감에 의존한 의사결정을 하게 된다
계약 직전에는 일정 압박, 대체 매물 부족, 정보 과부하 때문에  
정량적 판단보다 감에 의존하기 쉽다.  
따라서 계약 전 후보군을 걸러주는 **리스크 스크리닝 + 대안 추천**이 필요하다.

---

## 5. 솔루션 개요

이 서비스는 “좋아할 만한 집 추천”이 아니다.  
또한 최종 계약을 대신 결정해 주는 도구도 아니다.

> **현재 보고 있는 후보보다, 내 조건 안에서 상대적으로 더 안전한 전세 대안을 제안하는 1차 의사결정 지원 도구**

즉 추천의 목적은 취향 예측이 아니라 **보증금 회수 위험 최소화**이다.

---

## 6. 추천 시스템의 목표

1. 사용자의 현재 관심 후보보다 **보증금 회수 위험이 낮은 대안**을 찾는다.
2. 사용자의 보증금 규모를 기준으로 **예상 손실 노출 금액이 더 낮은 후보**를 우선 추천한다.
3. 단순히 가장 안전한 곳이 아니라, **실제로 살 수 있는 대안**을 추천한다.
4. 추천 결과와 함께 **왜 추천했는지**를 설명해 의사결정을 돕는다.

---

## 7. 추천 엔진의 기반 점수

추천 시스템은 기존 v3의 **구조 기반 리스크 스코어**를 핵심 ranking signal로 활용한다.

### 7-1. 기본 모델
| 지표 | 가중치 | 역할 |
|------|--------|------|
| 전세가율 | 50% | 깡통전세 구조 위험 |
| 전입전출 | 25% | 수요 감소 및 역전세 가능성 |
| 지하철 접근성 | 25% | 입지 방어력 |

### 7-2. 검증 철학
- HUG 사고 데이터는 **점수 입력값이 아니라 외부 검증용 정답셋**으로 사용한다.
- 점수는 사용자가 계약 전에 알 수 있는 변수로만 계산한다.
- 위험도가 높을수록 낮은 점수가 되도록 정규화한다.

### 7-3. 하락폭의 역할
하락폭은 추천 점수에 직접 넣지 않고,  
**손실 시뮬레이션 및 개인화 손실 노출 금액 계산**에 사용한다.

- 점수 = “위험한가?”
- 하락폭 = “얼마 잃을 수 있었나?”

---

## 8. 실거래가 데이터가 필요한 이유

현재 구조 기반 점수만으로도 추천은 가능하지만,  
**국토교통부 실거래가 데이터를 결합하면 추천의 현실성과 설명력이 크게 올라간다.**

### 8-1. 필요한 이유
- 현재 전세 보증금이 주변 실거래 흐름 대비 어느 수준인지 판단 가능
- 전세가율 계산의 기준이 되는 **매매 실거래가 최신성** 보강 가능
- 동일 생활권 내 **비슷한 가격대의 대체 후보**를 더 정확히 찾을 수 있음
- 특정 단지의 가격 급등·급락 구간을 포착해 손실 시뮬레이션을 더 설득력 있게 만들 수 있음

### 8-2. 추천 시스템에서의 역할
실거래가는 메인 점수 자체를 뒤집는 용도보다는 아래 용도에 적합하다.

1. **가격 기준선 생성**
2. **후보군 필터링 정교화**
3. **현재 후보 대비 추천 대안 비교**
4. **설명 카드 강화**

---

## 9. 추가해야 할 국토교통부 API

### 9-1. 필수 API
#### 1) 아파트 매매 실거래가 자료
- 용도: 전세가율 계산의 분모가 되는 **매매 실거래 기준가격** 확보
- 활용:
  - 동일 구/동/단지의 최근 매매 시세 산출
  - 최근 n개월 중앙값/평균값 산출
  - 전세 보증금 대비 매매 실거래 비율 계산

#### 2) 아파트 전월세 실거래가 자료
- 용도: 현재 입력 보증금이 **주변 전세 실거래 분포**에서 어느 위치인지 판단
- 활용:
  - 동일 생활권 전세 대안 후보군 추출
  - 비슷한 면적/유형의 전세 가격 범위 파악
  - 과도하게 높은 보증금 후보 탐지

### 9-2. 선택 API
#### 3) 오피스텔 매매 실거래가 자료
- 추천 범위를 오피스텔까지 넓힐 때 사용

#### 4) 오피스텔 전월세 실거래가 자료
- 오피스텔 전세/월세 대안 추천 시 사용

#### 5) 연립다세대 매매 / 전월세 실거래가 자료
- 아파트 외 주거유형까지 확장 시 사용

## 10. 입력 설계

### 10-1. 필수 입력
- **보증금 금액**
- **현재 관심 후보** (구 / 동 / 단지)
- **희망 지역 범위** 또는 생활권

### 10-2. 선택 입력
- **위험 성향**
- **면적대 / 평형**
- **주거 타입**
- **출퇴근 중심지**
- **손실 감내 한도**
  - 예: 예상 손실 3천만 원 초과 후보 제외

## 11. 위험 성향 설문 기반 개인화

이 시스템은 모든 사용자에게 동일한 추천을 주지 않는다.  
사용자의 위험 성향을 설문으로 파악한 뒤, 이를 추천 점수의 **패널티 강도와 선호 가중치**에 반영한다.

### 11-1. 위험 성향 카테고리
- **보수형**
- **중도위험형**
- **모험형**

### 11-2. 설문 목적
- 손실 회피 성향 파악
- 생활권/편의성 선호 강도 파악
- 현재 후보와 유사한 대안을 선호하는지 파악

### 11-3. 예시 설문 문항
1. 같은 조건이라면 조금 멀어져도 더 안전한 집을 선택하겠는가
2. 손실 가능성이 낮다면 편의성이 다소 떨어져도 괜찮은가
3. 현재 보고 있는 후보와 비슷한 조건이라면 약간의 위험은 감수할 수 있는가
4. 예상 손실 가능 금액이 큰 후보는 무조건 피하고 싶은가
5. 출퇴근 편의가 좋다면 다소 높은 전세가율도 고려할 수 있는가
6. 후보 간 차이가 작다면 안전한 쪽보다는 익숙한 지역을 선호하는가

각 문항을 1~5점 척도로 받고, 결과를 바탕으로 사용자 성향을 분류한다.

### 11-4. 분류 로직 예시
- 손실 회피 점수 높음 → **보수형**
- 안전성과 편의성 균형 → **중도위험형**
- 생활권/유사성 선호 높음, 일부 위험 감수 가능 → **모험형**

## 12. 추천 로직

추천 로직은 3단계 구조로 설계한다.

### 12-1. 1단계: Hard Filter
먼저 사용자가 실제로 고려 가능한 후보만 남긴다.

예:
- 보증금 ±10% 범위
- 희망 지역 또는 통근 가능권
- 원하는 주거 타입
- 최소 면적 조건
- 실거래 존재 여부 확인

이 단계의 목적은 “이론적으로 안전한 곳”이 아니라  
**실제로 대체 가능한 후보군**만 남기는 것이다.

### 12-2. 2단계: Safety Ranking
남은 후보를 구조 기반 리스크 점수와 손실 노출 기준으로 정렬한다.

핵심 반영 요소:
- 기본 모델 종합 점수
- 보증금 × 과거 하락폭 = 손실 노출 금액
- 전세가율
- 최근 매매 하락폭
- 주변 전세 실거래 대비 과열 여부

### 12-3. 3단계: Preference Re-ranking
마지막으로 사용자의 성향을 반영해 추천 결과를 미세 조정한다.

#### 보수형
- 손실 노출 패널티 강화
- 구조적 위험이 낮은 후보 우선
- 가격과열 후보 감점 강화

#### 중도위험형
- 안전성과 생활권/접근성 균형
- 가격 적정성과 편의성 균형

#### 모험형
- 안전성은 유지하되 입지/접근성 가중치 확대
- 현재 후보와의 유사성 반영 강화

---

## 13. 추천 점수 설계

### 13-1. 공통 안전점수
```text
안전점수 = 0.5*전세가율점수 + 0.25*전입전출점수 + 0.25*지하철점수
```

### 13-2. 개인화 추천점수
```text
추천점수 = 안전점수 - α*손실패널티 - β*가격과열패널티 + γ*생활권적합도 + δ*현재후보유사도
```

여기서 α, β, γ, δ는 위험 성향에 따라 다르게 설정한다.

### 13-3. 가중치 예시

#### 보수형
- α = 1.4
- β = 1.2
- γ = 0.8
- δ = 0.7

#### 중도위험형
- α = 1.0
- β = 1.0
- γ = 1.0
- δ = 1.0

#### 모험형
- α = 0.7
- β = 0.8
- γ = 1.2
- δ = 1.2

### 13-4. 설계 원칙
- 객관적 위험 점수는 공통으로 유지한다.
- 위험 성향은 최종 re-ranking에 반영한다.
- 절대 위험 기준을 초과하는 후보는 성향과 무관하게 제외한다.

즉 구조는 다음과 같다.

1. **Hard Filter**: 절대 위험 제거  
2. **Safety Ranking**: 공통 리스크 반영  
3. **Preference Re-ranking**: 성향 반영

---

## 14. 출력 설계

추천 결과는 단순 순위표가 아니라,  
**현재 후보 대비 얼마나 개선되는지**를 보여줘야 한다.

### 추천 카드 예시
#### 추천 1. 양재동 A단지
- 현재 후보보다 종합 점수 **+12점**
- 최근 6개월 매매 하락폭 **3%p 낮음**
- 입력 보증금 기준 손실 노출 **2,300만 원 감소**
- 최근 전세 실거래 중앙값과 유사
- 강남 생활권 유지

### 설명 문구 예시
- 현재 후보 대비 전세가율이 낮습니다.
- 최근 실거래 기준 매매 가격 방어력이 더 높습니다.
- 같은 보증금 구간에서 손실 노출 가능성이 더 낮습니다.

## 15. 출력 유형

### 15-1. 가장 안전한 대안
- 손실 노출 최소화 중심
- 보수형 사용자에게 적합

### 15-2. 가장 균형 잡힌 대안
- 안전성과 생활권을 함께 고려
- 일반 사용자에게 적합

### 15-3. 현재 후보와 가장 비슷하지만 더 안전한 대안
- 현재 관심 후보를 크게 벗어나지 않는 후보
- 사용자의 심리적 전환 비용을 줄여줌

---

## 16. 추천 사유와 제외 사유 설명

### 16-1. 사용자에게 보여줄 핵심 설명
- 왜 현재 후보보다 더 안전한지
- 어느 지표에서 개선되는지
- 내 보증금 기준 손실이 얼마나 줄어드는지
- 생활권이나 접근성을 얼마나 유지하는지

### 16-2. 제외 사유 예시
- 예산은 맞지만 손실 노출이 높아 후순위
- 종합 점수는 높지만 생활권 조건에서 제외
- 전세가율이 높아 구조적 리스크가 커서 제외

이 기능은 사용자가 결과를 **블랙박스**로 느끼지 않게 해준다.

---

## 17. 데이터 파이프라인 제안

### 17-1. 수집
- 국토교통부 아파트 매매 실거래가 API
- 국토교통부 아파트 전월세 실거래가 API
- 기존 내부 데이터
- HUG 사고 검증 데이터

### 17-2. 적재
- Raw 테이블: API 원천 응답 적재
- Cleansed 테이블: 법정동 코드, 계약월, 단지명, 면적, 가격 정규화
- Feature 테이블:
  - 최근 3개월 매매 중앙값
  - 최근 3개월 전세 중앙값
  - 전세가율
  - 하락폭
  - 후보 유사도
  - 위험 성향별 추천 가중치
- Serving 테이블:
  - 추천 후보 리스트
  - 추천 설명용 피처
  - 비교 카드용 집계값

### 17-3. Serving
- 사용자 입력 기반 후보 필터링
- 리스크 점수 계산
- 성향 설문 결과 매핑
- 추천 랭킹 생성
- 추천 사유 / 제외 사유 생성

---

## 18. 탭 구조

### 탭1: 전세 안전 진단
- 현재 후보 위험 진단
- 종합 등급과 해설 제공

### 탭2: 위기 시뮬레이션 + 개인화 대안 추천
- 과거 하락 시 손실 노출 계산
- 비슷한 예산대에서 더 안전한 후보 추천
- 추천 이유와 개선 포인트 표시
- 위험 성향 기반 추천 순위 조정

즉, 추천 시스템은 단독 기능이라기보다  
**위험 진단 이후 사용자의 다음 행동을 안내하는 핵심 기능**으로 설계된다.

## 19. 데모 플로우 예시

### 사용자 입력
- 현재 보고 있는 후보: 관악구 A단지
- 보증금: 5억
- 생활권: 강남 출근권
- 위험 성향 설문 결과: 보수형

### 시스템 동작
1. 4.5억~5.5억 후보 추출
2. 강남 접근 가능 지역 필터
3. 구조 기반 리스크 점수 계산
4. 손실 노출 금액 계산
5. 실거래가 기반 가격 적정성 평가
6. 위험 성향 기반 re-ranking
7. 현재 후보보다 더 안전한 후보 랭킹

### 출력
- 추천 1: 양재동 B단지
- 추천 2: 문래동 C단지
- 추천 3: 방배동 D단지

### 추천 사유
- 현재 후보보다 전세가율이 낮음
- 손실 노출 금액 감소
- 구조 기반 리스크 점수 우위
- 생활권 유지
- 사용자 성향 기준으로 손실 회피 우선 반영

## 20. 기대 효과

### 사용자 관점
- 위험한 후보를 빠르게 걸러낼 수 있다
- “그럼 어디가 더 안전한가?”에 바로 답을 얻을 수 있다
- 내 보증금 기준으로 현실적인 손실 노출을 이해할 수 있다
- 내 성향에 맞는 추천을 받을 수 있다

### 제품 관점
- 단순 진단 앱에서 **의사결정 지원 제품**으로 진화한다
- 분석 도구가 아니라 실제로 선택을 돕는 제품으로 보인다
- 개인화 추천 시스템으로 확장성이 높다

### 해커톤 관점
- 점수 산출 + 백테스팅 + 실거래가 + 성향 기반 개인화 추천까지 연결되어 완성도가 높다
- “위험하다”에서 끝나지 않고 “대안을 제시한다”는 점이 강하다
- AI / 데이터 / 제품 스토리텔링이 자연스럽게 연결된다

## 21. MVP 범위 제안

### MVP 1단계
- 서울 아파트만 대상
- 아파트 매매 실거래가 API
- 아파트 전월세 실거래가 API
- 구/동 단위 추천
- 위험 성향 3분류
- 최근 3~6개월 기준 비교

### MVP 2단계
- 오피스텔 추가
- 단지 단위 정교화
- 생활권 점수 강화
- 추천 설명 자동화 고도화

### MVP 3단계
- 연립/다세대 확장
- 계약 체크리스트 연동
- 보증보험 가입 가능 여부 확인 흐름 연결

## 22. 구현 시 유의사항

- 공공데이터포털 API는 서비스키 발급이 필요하다.
- REST/XML 응답을 파싱하고 정규화하는 레이어가 필요하다.
- 법정동 코드와 계약년월 단위 조회 구조를 기준으로 배치 수집 설계가 적합하다.
- 실거래 데이터는 운영상 **월 단위 적재 + 주기적 동기화** 전략이 안정적이다.
- 위험 성향은 절대적 진단값이 아니라 **추천 조정 변수**로 사용하는 것이 바람직하다.
- 모험형 사용자라도 절대 위험 기준을 넘는 후보는 추천에서 제외해야 한다.
